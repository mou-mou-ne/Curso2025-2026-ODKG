Resource Naming Strategy

Purpose

This document explains the URI and resource naming strategy for the BECA dataset. Since the dataset is quite small, hash URIs will be used (see Sauermann & Cyganiak, 2008).

The URI domain to be used is: https://iamabsa.github.io/BECA/chargingstation##12345/. This domain is governmental and follows the data publisherâ€™s identity. A subfolder BECA indicates that the ontology and data are related to the BECA Example.

Ontology Model Classes

  • Classes follow the pattern: 
    https://iamabsa.github.io/BECA/ontology/<ontologyName>#<className>
  • Example: Charging station class: 
    https://iamabsa.github.io/BECA/ontology/ChargingOntology#ChargingStation

Ontology Model Properties

  • Properties follow the pattern: 
    http://iamabsa.github.io/BECA/ontology/<ontologyName>#<propertyName>
  • Example (the hasPower property of ChargingStation): 
    http://iamabsa.github.io/BECA/ontology/ChargingOntology#hasPower

Instances

  • Instances follow the pattern: 
    https://iamabsa.github.io/BECA/resource/<className>#<identifier>
  • Example (an instance of a charging station in Vienna): 
    https://iamabsa.github.io/BECA/resource/ChargingStation#WIEN1234
