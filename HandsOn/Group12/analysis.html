Resource Naming Strategy

  • Since the dataset is quite small, hash URIs will be used (see Sauermann & Cyganiak, 2008).
  • The URI domain to be used is https://iamabsa.github.io/BECA/chargingstation##12345 /. This domain is chosen because it is a governemental and it follows the data publisherâ€™s identity. We could've also used the governemental one but since the transformed RDF data and ontology can be published online, it is better to use "our own".
  • A subfolder BECA will be added to the domain to indicate that the ontology model and data are related to the BECA Example (because we are truthully using our teacher's example as a guideline as you probably already noticed).

Ontology model classes

Classes in the ontology will follow the pattern:

https://iamabsa.github.io/BECA/ontology/#;

Example: The charging station class :

https://iamabsa.github.io/BECA/ontology/ChargingOntology#ChargingStation

Ontology model properties

Properties will follow the pattern:

http://iamabsa.github.io/BECA/ontology/<ontologyName>#<propertyName>

Example (the hasPower property of the chargingstation class):

http://iamabsa.github.io/BECA/ontology/ChargingOntology#hasPower

Instances

Instances (the actual data entries (charging stations (parenthesis-ception))) follow the pattern:

https://iamabsa.github.io/BECA/resource/#;

Example (an instance of a dwelling, e.g., in Belgrade):

https://iamabsa.github.io/BECA/resource/ChargingStation#WIEN1234
